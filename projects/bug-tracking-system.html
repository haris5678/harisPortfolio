Bug Tracking System

A Full-stack application developed using React and Ruby on Rails.

Project Overview

The Bug Tracking System project was full-stack application built using React and Ruby on Rails. It serves as a comprehensive bug management solution, allowing users to sign up, log in, and log out.
Managers and users can create and participate in multiple projects, while bugs are associated with specific projects. Each bug has a unique title, a creator, and a developer assigned to resolve it. The bugs can be classified as either "feature" or "bug" types and have a status of "new", "started", or "completed" for features, or "new", "started", or "resolved" for bugs.
Screenshots, descriptions, and deadlines can be attached to bugs, with file uploads handled by ActiveStorage. The system also incorporates role-based authorizations using the Pundit Gem, granting managers the ability to create, edit, and delete projects, and add or remove developers and QAs. Developers can assign bugs to themselves and mark them as resolved, while QAs can report bugs for all projects and view all projects.
The Bug Tracking System project demonstrates proficiency in React, Ruby on Rails, file uploads, and role-based authorizations, offering a powerful tool for efficient bug tracking and project management.
